# 阶段十八规划 · 四视角锐评 + 迭代计划(v10.4 → v11.0)

> 基线:**v10.3.6**(2026-06-09)· tsc 0 / vitest 2140 / playwright 42(smoke + axe a11y + reduced-motion + keyboard)
> 方法:产品经理 / AI 漫剧从业者 / 设计师 / 全栈架构师 四视角锐评 → 可行性方案 → 逐版本计划。
> 所有批评先经代码事实核验(已建成的不重复立项:`continuity.ts`、`consistency-report.ts`、`mckee-skill.ts`、`drama-tropes.ts`、`budget-guard/enforce.ts`、9:16 已在 export/distribution 支持)。
>
> 竞品核验(2026-06-09 联网):Kling v3/3.0 登顶 text-to-video arena,Veo 3.1 综合最强(48kHz 原生音频 + 4K),Seedance 2.0 Fast 性价比第一(~$0.022/s),Runway Gen-4.5 I2V 最强——README 竞品表**无需更新**;生成层已原生多镜/口型/4K → 「护城河在制作/平台层」的定位判断继续成立,且更紧迫。

---

## 一、锐评

### 1. 产品经理视角

| 级别 | 问题 | 锐评 |
|---|---|---|
| **P0** | **激活漏斗断裂(Time-to-Wow > 30 分钟)** | 无 key 全是占位图。新用户旅程 = clone → 配 LLM+图像+视频 3 把 key → 才见第一个真镜头;而 SaaS 竞品 10 秒出片。开源自托管产品的等价解不是"也做 SaaS",而是:**演示工程一键导入**(预生成资产,0 key 可浏览完整成片工作台)+ **一把 key 分级体验**(只配 LLM → 剧本/分镜/审计全真,画面如实占位)。 |
| P1 | **定位与首页卖点漂移** | 竞品分析已三次得出结论:生成层不是我们的仗(Kling 3.0 原生多镜+口型,Veo 3.1 原生音频)。但首页 hero 仍在卖「从灵感到成片」的生成叙事。应改卖**制作台**:节奏/麦基审计、质量门禁、连续性、多集编排、AAF 进剪辑线、团队协作——生成层明示为「接入当下最强引擎(BYO)」。 |
| P1 | **留存面缺失** | 用户没有回访理由:无「继续创作」入口、无项目健康摘要、无周报 digest。通知系统已有(SSE),缺的是内容。 |
| P2 | **模板市场冷启动** | 机制全(存/评分/收藏/一键起片)但无内容运营策略;18 个题材模板是底仓,缺种子内容计划与「官方精选」位。 |

### 2. AI 漫剧从业者视角

| 级别 | 问题 | 锐评 |
|---|---|---|
| **P0** | **竖屏短剧不是一等公民** | 9:16 在导出/分发层已支持,但**创作默认 16:9**,分镜构图 prompt、字幕安全区、封面候选全按横屏设计。2026 短剧主战场是 9:16(抖音/快手/TikTok/ReelShort 系)。竖屏要从「能导出」升级为「项目级优先模式」。 |
| **P0** | **一致性纵深不够** | 锁脸≤3 + 风格圣经 + DNA + `continuity.ts` 是好底子,但从业者真实痛点在**资产级连续性**:同一件外套跨 30 镜不变色、同一房间道具摆位不漂移。需要「场景资产台账」(道具/服装/场景登记 + 跨镜引用)+ 漂移自动质检(Vision BYO,无 key 走启发式)。 |
| P1 | **钩子指标缺平台化落地** | 叙事理论侧已强(mckee-skill / drama-tropes / 节奏审计),但平台运营指标缺位:**开场 3 秒留人钩子分、集尾悬念分、BGM 卡点对齐率**。这三个数字是短剧投流团队每天看的,审计报告里应该有。 |
| P2 | **配音 retake 工作流** | voice-shelf/overrides 已有,缺台词级情绪标注 + 重录队列 + 版本对比试听(配音导演的日常)。 |

### 3. 设计师视角

| 级别 | 问题 | 锐评 |
|---|---|---|
| **P0** | **双设计语言边界失控** | cinema-theme(琥珀 #C9A35E·衬线·mono)与默认暗金(#E8C547)并存。v10.3.3 的真 bug(金色 CTA 在非 cinema 页透明)就是这笔债的利息——var() 兜底已打补丁,但**根治需要**:共享 token 层 + 「页面↔语言」归属清单 + 约定(cinema-* 类要么只在 .cinema-page 用,要么必须带字面兜底)。 |
| P1 | **创作工坊首跑认知过载** | 多参货架/运镜/锁脸/LoRA 库/模板库一次全摊开。需要渐进披露:首跑三步引导(写创意→选风格→ROLL)+「简易/专业」模式开关(localStorage 记忆)。 |
| P1 | **移动端定位不明** | e2e 移动端只验「渲染不崩 + a11y」。380px 上三栏工作台必然降级。应明确**移动 = 看片/评论/审批**,并把这个定位做成体验(移动项目页强化预览+评论,创作页给「去桌面」引导),而不是假装全功能响应式。 |
| — | a11y | 本阶段已成强项(对比度全站 AA、axe 16 断言、reduced-motion、键盘焦点+陷阱),**保持门禁即可,不立新项**。 |

### 4. 全栈架构师视角

| 级别 | 问题 | 锐评 |
|---|---|---|
| **P0** | **流水线跑在 HTTP 请求生命周期里** | create-stream 在 API route 内做分钟级长任务,部署重启/超时 = 进度丢失。这是上生产**最大单点**。代码里已有现成的阶段抽象(pipeline-stages 四环节 + 重跑下游),缺的是把执行从 route 挪进**任务表 + worker**。 |
| **P0** | **JWT 存 localStorage** | XSS = 直接丢 token。应迁 httpOnly cookie(SameSite=Lax)+ 服务端双读兼容(cookie 优先、Bearer 兜底,E2E mint 与 SSE client 不破)。 |
| P1 | **实时只支持单实例** | event-bus 是 globalThis EventEmitter,多副本部署时评论/通知互不可见。bus 后加适配器:BYO `REDIS_URL` → pub/sub,无则回退进程内(现状零损)。 |
| P1 | **产物存储无对象存储层** | lipsync local-2d 直接产 `data:video/mp4`(base64 入库!),资产落本地盘。需要 storage adapter(local-disk 默认 / S3 兼容 BYO),lipsync 改落文件返 URL。 |
| P2 | **e2e 缺用户旅程** | 42 条全是渲染/a11y/键盘门禁,没有「登录→建项目→ROLL→导出」主链路。用 mock provider(确定性假图/假视频)一条 journey spec 即可,且它是后续大重构的安全网。 |
| P2 | **可观测性** | provider 遥测已有,缺结构化日志(console JSON)+ 可选错误上报 DSN。自托管友好优先。 |

## 二、迭代计划

**原则**:每版独立验收(tsc / vitest / e2e 三绿 + 各自验收标准);先建安全网再动承重墙;架构大件拆小步;延续「确定性启发式 + BYO 真模型」哲学。

### 阶段十八 A · 生产级地基(v10.4.x)— 架构师主导

| 版本 | 交付 | 规模 | 验收标准 |
|---|---|---|---|
| v10.4.0 | **mock provider + journey e2e(安全网先行)**:确定性假图/假视频/假 TTS provider(注册进既有 provider registry);e2e:登录→建项目→ROLL→出片→导出 | S | journey spec 双端绿;mock 下全链路 < 60s |
| v10.4.1 | **任务表 + worker**:`pipeline_jobs`(id/type/step/state/payload/attempts)+ 进程内 worker tick;create-stream 改投递,SSE 进度读 job 状态 | M | `kill -9` dev server 重启后任务续跑;journey e2e 不变绿 |
| v10.4.2 | **幂等续跑 + 死信**:step 幂等 key(项目+镜头+阶段),重复执行先查产物;失败退避重试 N 次 → dead-letter 列表(UI 可见可重投) | M | 模拟 step 中途崩溃:重启后不重复生成/不重复计费 |
| v10.4.3 | **auth 加固**:登录签发 httpOnly cookie;`getUserFromRequest` 双读(cookie→Bearer);logout 清 cookie;E2E mint 改 addCookies | S | 删除 localStorage token 后会话仍在;42+journey e2e 全绿 |
| v10.4.4 | **storage adapter**:`lib/storage.ts`(local-disk 默认 / S3 兼容 BYO);lipsync data:URI 改落盘返 URL;新产物走 adapter | S | 配 MinIO 冒烟通过;无配置行为与现状一致 |
| v10.4.5 | **event-bus 适配器**:`REDIS_URL` 存在 → redis pub/sub,否则进程内;接口零变化 | S | 双进程起两个 dev,A 发评论 B 收到 SSE |

### v11.0 里程碑(收口)

阶段十八 A+B+C 全绿后合并大版本:生产级部署文档(单机/多副本两种拓扑 + 环境变量矩阵)+ 设计 token 统一收尾(cinema/default 边界清单)+ 全量回归 + 竞品表整轮联网刷新。

## 三、排序依据 / 风险 / 非目标

**为什么 A 先**:任务队列是后续一切长任务功能(批量整季、4K 重渲队列、retake 队列)的承重墙;auth/storage 是上生产硬门槛。B 的激活改造收益最高但依赖「mock provider」(v10.4.0)才能做出可信的占位体验。

**关键风险**:v10.4.1–4.2 动 create-stream 主链路 → 对策:v10.4.0 安全网先行,重构期间 journey e2e 必须每版全绿;任何一步可独立回滚(任务表不删旧路径,feature flag `PIPELINE_QUEUE=1` 灰度一版后切默认)。

**非目标(本阶段明确不做)**:自研/微调生成模型(生成层军备竞赛不参战);移动端创作(定位为看片/评论);Yjs 协同升级(现状够用);K8s operator/Helm(单机+多副本文档先行)。
