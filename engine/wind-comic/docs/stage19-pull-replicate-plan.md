## 一、对标拆解:OiiOii「拉片复刻」到底是什么

三段闭环(摘自文章实测):

1. **拉片(拆解)**:上传一条视频 → 自动拆成逐镜五栏结构化表:
   叙事要素(场景/角色/台词对白)· 时间(时长/起止,毫秒级)· 镜头语言(景别/构图/
   镜头类型/运镜方法/焦距与景深)· 影像处理(光影与色调/剪辑事件/数字颗粒)·
   声音(音乐音效/分镜功能/镜头叙事功能),配每镜可播放缩略图。
2. **替换**:用户只说要换什么 —— 角色/场景/道具/服饰/logo 全可换,文字指令或参考图
   (上传 / 站内资产)都行;Agent 自动改写逐镜提示词,**批量**完成,不用一镜一镜调。
   提示词全开放可编辑;分镜参考图自动匹配。图片越多解析越久(用户预期管理)。
3. **复刻**:按原片镜头结构 + 替换后的 prompt **并行**生成各镜 → 合成新片。
   还原度覆盖剧情/画面/镜头语言/UI 排版/调色打光/台词音效。

文章点出的真实价值:「研究 & 复刻一条爆款」的前期分析 + 改写流程被压缩掉;
副产物 —— 自动生成的提示词本身质量高,可当学习材料。

## 二、我们的独特优势(方案据此设计)

1. **出厂参数优势**:竞品拉自家生成的片要"AI 看图猜参数";本项目流水线生成时就持有
   全部真实参数 —— `ScriptShot` v2.8 摄影语言字段(shotSize/lens/cameraAngle/
   cameraMovement/lightingIntent/composition/editPattern/diegeticSound/scoreMood/
   rhythmicSync/whyThisChoice)+ duration/dialogue/emotion,与五栏几乎一一对应。
   自家项目的拉片表是**真值**,零生成成本。
2. **复刻管线免费**:替换后的逐镜 prompt 灌回自家流水线即可 —— 剧本改编路径
   (parsedScript,跳过创意环节)是现成注入点;并行生成(PIPELINE_QUEUE)、
   逐镜重生(rerun/regenerate-shot)、参考图一致性(cref/sref)、合成(video-composer)全部已有。
3. **拉完即用闭环**:竞品拉完只能看或单次复刻;我们可以「拉片 → 替换 → 复刻」之外
   再加「存为私有模板」,把爆款结构沉淀成可复用资产(接既有模板市场)。

## 三、版本计划

### v11.1.0 — 拉片表(确定性地基,自家项目)【S】
- `lib/pull-sheet.ts`:定义 **PullSheet schema**(五栏 × 逐镜,即全阶段统一数据结构)
  + `buildPullSheetFromScript(script, assets)` 纯函数:从 script_data + storyboard/video
  资产构建拉片表(时间轴 = duration 累计;缩略图 = 资产 URL)。
- 项目页新「拉片」tab:五栏表 UI(对齐截图信息密度,cinema 体系)+ CSV 导出。
- 演示项目《雨夜信号》即开即见。
- 验收:演示工程拉片表五栏全有值;CSV 导出可开;单测覆盖 build 函数。

### v11.1.1 — 外部视频拆条 + 拉片(BYO Vision)【M】
- 上传外部视频(或贴 URL)→ `ffmpeg` 场景切分(scene detect,复用 beat-detect 的
  ffmpeg 设施)→ 逐镜:起止/时长/缩略图/末帧抽取(last-frame-extractor 复用)。
- 零配置:确定性骨架表(切点/时长/缩略图)诚实标注「镜头语言等需配 Vision key」。
- 配 Vision key:逐镜抽帧 → Vision LLM 按 PullSheet schema 打标(llm-client 范式,
  失败降级骨架);打标进度走 pipeline_jobs(type='pull-sheet')。
- 产物存 project_assets type='pull-sheet'(独立「拉片工作台」入口,不挂具体项目也可用)。
- 验收:mock 模式骨架表全链可跑(e2e);打标 schema 校验单测。

### v11.1.2 — 替换 + 复刻(杀手锏)【M】
- 拉片表 → 「替换工作台」:全局指令(如「所有人物换成猫」)或逐维度替换
  (角色/场景/道具/服饰),支持参考图(上传 / 角色库 / Cameo 资产)。
- prompt 改写:LLM 把「原镜头描述 × 替换指令」批量改写为新逐镜 prompt
  (规则兜底:确定性文本替换);**所有改写后 prompt 开放可编辑**(对齐 OiiOii,
  也吃到「提示词可学习」的副产物价值)。
- 复刻起片:构造 shots(镜头语言字段从拉片表回填 v2.8 字段)→ 走剧本改编路径
  起片(跳过 Writer 创意)→ 并行生成 → 合成。镜头时长按拉片表锁定。
- 「存为私有模板」:拉片结构(structureHint + 逐镜 cameraWork)沉淀进模板市场。
- 验收:demo 用例 —— 对一条演示视频拉片 → 全员替换指令 → mock 引擎下复刻出
  同结构新片(镜数/时长一致,资产 URL 全新);e2e 全链。

## 四、风险与边界(诚实清单)

- **版权敏感**:上传他人视频复刻有侵权风险。产品定位为「结构学习 + 主体替换的二次创作」,
  不做逐帧复制;UI 明示「请确认你对参考素材的使用权;复刻产物为同结构新内容」。
- **Vision 打标成本**:逐镜抽帧 × Vision 调用,图多解析久(文章原话同款预期管理)——
  打标走队列 + 进度推送,镜数上限护栏(如 ≤60 镜)。
- **场景切分精度**:ffmpeg scene detect 对转场柔和的片会漏切/过切 —— 工作台支持
  手动合并/拆分镜头(确定性兜底,人工可校准,与台账/钩子审计同款人机协作哲学)。
- **非目标**:不做音频指纹级 BGM 识别;不做原片素材直接抽取复用(只生成新内容)。
