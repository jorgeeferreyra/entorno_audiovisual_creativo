## 五. 长描述 (适合 ModelScope 的"详细介绍"区, 500-1000 字)

### 中文版

**Wind Comic 是什么**

Wind Comic 是一条**开源的多 Agent AI 短剧生成流水线**. 一句创意 (例如"重生归来的霸总当街拆穿前未婚妻的婚礼骗局") 输入, 8 个 AI agent 各司其职逐步产出: 剧本 → 角色三视图 → 场景概念图 → 多镜分镜 → 视频成片 → 配音 + BGM + 中文字幕 → 最终 mp4. 全过程在你自己的机器上, 可选接 Minimax / Veo / Kling 等图像 / 视频引擎, LLM 端 OpenAI 兼容 (DeepSeek / Claude / 通义 / Kimi / 本地 Ollama 都行).

**为什么不是又一个 "AI 视频" 工具**

主流 AI 视频工具 (Sora, Kling, Vidu, Runway) 都是"输入 prompt, 输出短片"的黑盒模型. 痛点:
- 一段连续镜头, 没法做"6 镜短剧"
- 角色跨镜变脸
- 画风每镜不同
- 中文字幕一律乱码
- 无法实时多人协作
- 厂商锁定, 你的 prompt 喂别人的模型

Wind Comic 走相反路线: **流水线 + 多 Agent + 严格契约**. 每个 agent 是专家, 用结构化数据 (而非随便的文本) 交接. 编剧只关心 McKee 三幕和短剧 trope; 角色师只关心三视图和 DNA 抽取; 分镜师只关心 vision 审计和画风一致性. 任何一环出问题, 我们都有"重生 / 兜底 / fallback"机制.

**核心技术亮点**

1. **Style Bible Frame** — 导演规划完事后, 先渲染一张 canonical "key art" 帧, 然后作为所有后续分镜的首位 `--sref`. 等于"先定海报, 再拍片", 关掉 60% 画风跳脱.

2. **Character DNA + Cameo Vision Retry** — 三视图过 Vision LLM 抽 8 维结构 (眼型 / 下颌 / 鼻型 / 嘴型 / 发型 / 发色 / 肤色 / 标志服饰), 作为自然语言锚点注入每个出场镜头. 加 cameo-vision-retry: 某镜角色匹配度 <75 自动重生, cw 拉高重画. 实测 face match 平均 82 分, 业内 cref+sref 平均 65 分.

3. **真正能用的中文字幕** — AI 视频模型对 CJK 字符全部画成鬼画符. 我们的做法: 从视频 prompt 剥掉对白文字 + 加狠负向 prompt (`--no text --no chinese --no captions --no calligraphy`) 让模型只画"角色在说话"动作, 然后后期 ffmpeg `subtitles` filter + libass + 系统 CJK 字体 (PingFang/Noto Sans CJK) 真烧字幕.

4. **Logic Pro 风格多轨道时间线** — 3 轨布局 (分镜/BGM/字幕), 拖拽改时间, 双击改字幕文字, 拖边沿改时长, 自动吸附邻居, 真 BGM 波形 (Web Audio API decode). 加一层**实时多人协作**: Yjs CRDT, 你能看到其他用户的鼠标光标 + 名字标 + 头像下方显示对方在哪个 tab, Y.Map 锁防止两人编辑同段.

5. **接你自己的 LLM** — 整条流水线对 LLM 无绑定. 改 3 行 `.env` 就能换 provider: gpt-4o / Claude (via OpenRouter) / DeepSeek-r1 / 通义 Qwen-Max / 智谱 GLM-4.5 / Moonshot Kimi / 本地 Ollama. **0 改代码**. Reasoning 模型 (DeepSeek-r1, MiniMax-M2, o1/o3) 自动检测 + 拉超时 + 剥 `<think>` 块.

### English version

**What is Wind Comic?**

Wind Comic is an **open-source multi-agent AI pipeline for short-form drama generation**. Input one sentence (e.g. "A reborn CEO confronts his cheating ex-fiancée at her wedding"), 8 specialized AI agents pass off in sequence: script → character turnarounds → scene concept art → multi-shot storyboards → video → voiceover + BGM + Chinese subtitles → final mp4. Runs on your own machine. Plugs into optional providers (Minimax / Veo / Kling / etc.) for images and video. The LLM side is OpenAI-compatible (works with DeepSeek / Claude / Qwen / Kimi / local Ollama out of the box).

**Why not just another "AI video" tool**

Mainstream tools (Sora, Kling, Vidu, Runway) are all "prompt-in, clip-out" black boxes:
- One continuous clip, can't do 6-shot narratives
- Characters change faces between shots
- Visual style drifts
- Chinese subtitles always render as garbled glyphs
- No real-time multi-user editing
- Vendor lock-in for the LLM

Wind Comic takes the opposite approach: **pipeline + multi-agent + strict contracts**. Each agent is a specialist passing structured data (not free-form text) to the next. The Writer only thinks about McKee 3-act and short-drama tropes; the Character Designer only thinks about turnaround sheets and DNA extraction; the Storyboard Artist only thinks about vision audits and style coherence. Every stage has retry / fallback / regen built in.

**Top technical highlights**

1. **Style Bible Frame** — render one canonical key-art frame after the Director plan, then pass it as the first `--sref` for all subsequent storyboard shots. Closes ~60% of "looks like 6 different shows" drift.

2. **Character DNA + Cameo Vision Retry** — turnaround sheet → Vision LLM extracts 8-dim structure (eye shape, jaw, nose, mouth, hair style/color, skin tone, signature outfit) → injected as natural-language anchor in every shot prompt. Combined with cameo-vision-retry: any shot scoring <75 on face match auto-regens with boosted cw. Face match averages 82 (industry cref+sref ~65).

3. **Real CJK subtitles** — AI video models render garbled glyphs for Chinese. Our fix: strip dialogue text from video prompt + aggressive negative prompts (`--no text --no chinese --no captions --no calligraphy`) so the model only generates lip motion → bake real subtitles in post via ffmpeg `subtitles` filter + libass + auto-discovered system CJK font (PingFang/Noto Sans CJK).

4. **Logic Pro-style multi-track timeline** — 3 tracks (shots / BGM / subtitle), drag to retime, double-click to edit subtitles, drag edges to resize, auto-snap to neighbors, real BGM waveform (Web Audio API decode). Plus **real-time multiplayer** via Yjs CRDT: live cursors with name labels, presence chips showing each collaborator's current tab, Y.Map segment locks preventing two users editing the same segment.

5. **Bring your own LLM** — zero vendor lock. Edit 3 `.env` lines to swap to gpt-4o / Claude (via OpenRouter) / DeepSeek-r1 / Qwen-Max / GLM-4.5 / Kimi / local Ollama. **Zero code change.** Reasoning models (DeepSeek-r1, MiniMax-M2, o1/o3) auto-detected with extended timeout and `<think>` block stripping.

**Who it's for**

- **Vertical drama creators**: 1 idea → 30s polished short, consistent characters, burnt subtitles, pacing audit
- **Content marketing teams**: brand consistency (Style Bible) + real Chinese subtitles + collaborative editing
- **Comic/manhua studios**: script → storyboards in your art style, triple-locked character consistency, per-shot regen with reference image upload
- **Indie filmmakers**: Logic Pro-style timeline + real BGM waveform editor
- **AI open-source builders**: study multi-agent patterns / fork / use your own LLM provider
- **Privacy-sensitive deployments**: fully self-hostable + local Ollama, scripts never leave your machine
